# 《不存在的住户》全流程与完美通关

> 完整剧透。本文既是走查清单，也是修改进度门槛时的开发参考。

## 流程总览

```text
开场记忆与醒来
  → CJ-0713登录
  → 1204空置房与失联儿童
  → 1304事故、顾小满与顾长河残留会话
  → 1104周明川遗体与注销账号
  → 恒目流程证据
  → 1404投诉与主角身份
  → 记忆覆盖反制
  → 下班：提交全部证据 / 重新进入循环
```

游戏故意要求玩家搜索怀疑的关键词，而不是按固定目录顺序阅读。下文给出的是保证不漏证据的完美路线，不代表UI应把它显示成任务清单。

调查台账按房间归并为四个章节。对应事实推导完成前，台账和推导目录只显示房间号；完成后才揭示章节名：`空房间与隐形孩子`、`没离开的人`、`周明川，最后一次呼叫`、`明天，再一次`。

房号是入口词，不是全案目录。直接搜索时，索引应保持以下范围：

| 房号 | 房号检索的最大范围 |
| --- | --- |
| 1204 | 夜间投诉、空置巡检、顶面渗漏排查 |
| 1304 | 顶面渗漏排查、顾长河表层回访、关联的1204投诉；许芷遥获救后新增墙面修补前影像 |
| 1104 | 1304章节完成前不显示内部材料；读取周明川同步摘要后，房号检索新增房间实况、同步摘要与墙体联合记录 |
| 1404 | 前期最多出现一条破碎关怀索引；最终章节只增加投诉工单，不列出后续身份档案 |

服务排班、公开通报、儿童身份、事故附件、异地证明、四份1404加密材料等都必须用上游正文给出的二级词找到。

## 0. 开场与登录

1. 观看三段日常记忆：厨房晚饭、雨中等车、周末整理衣物。
2. 文案从日常过渡到生命脆弱和意识错位，随后从梦境醒来。
3. 点击进入身份认证终端。
4. 使用工牌进入，或选择密码登录：`CJ-0713` / `0713`。
5. 登录瞬间短暂飘红闪烁，然后进入正常的 `物业管理系统`。
6. 全新存档首次进入工作台后，首页先保持约3秒安静，再播放一次留言提示音并弹出初始信息窗。继续游戏、刷新恢复和后续剧情留言不使用这段首次登录延迟。

验证点：

- 开场不是营销首页，第一屏就是演出。
- 开场、普通登录与结局使用忧伤阴郁的D小调主题。正常进入物业系统后不播放背景音乐，只保留业务提示音；玩家查到物业单眼标记的异常来源并收到“检索行为告警”后，才启用 Alexander Nakarada 的悬疑氛围曲 `Countdown`（CC BY 4.0，来源见 `public/audio/SYSTEM_MUSIC_SOURCE.md`）。暗红403警告、周明川账号登录、本地档案和身份侦测阶段改用 Rafael Krux 的惊悚氛围曲 `Lights - Creepy Ambient Suspense`（CC BY 4.0，来源见 `public/audio/HORROR_MUSIC_SOURCE.md`）。音乐在首次用户操作后开始，静音偏好跨刷新保留。
- 播放监控视频或四轨声纹样本时，背景音乐自动压低，不能遮蔽案件原始音频。
- 新证据实际写入调查台账时播放一次独立的低音确认提示；重复核验、刷新和恢复旧存档不得重复播放。
- 刷新在hash路由和本地存档允许的范围内恢复当前页面。
- 登录页的 `遗忘` 二次确认后清除全部本机进度并回到初始状态。
- 未恢复周明川凭据之前，账号选择中不能直接使用 `ZM-0602`。

## 1. 1204：从投诉到儿童救援

### 1.1 打开起始工单

打开 `1204 夜间滴水投诉复核`，记住：

- 每天 `00:04` 开始，`00:10` 结束，共六分钟。
- 1304近24小时用水为零。
- 报修人怀疑楼上有人，但系统不能直接替玩家核验其身份。

首页的“待办工单”只显示系统直接派发的当前工单，或玩家已经找到且需要填写、提交、编排或核验的回执。1204起始工单会持续保留到实际占用核验回执建立，1404最终工单也会在玩家寻找首份受限档案期间保留，避免玩家打开一次后失去业务入口。普通证据文章、受限档案及其检索词不会被逐篇列为“下一份档案”；没有正式工单或待填回执的阶段显示“暂无待填回执”，由玩家根据已阅记录自主检索。已出现的回执可从待办卡返回。留言回复、真相推导和注销账号登录等明确的系统处置节点仍可临时占用同一张待办卡。

可以先查看留言板。林若岚、许先生、0702和1302的留言混合了线索与噪声。留言按发信人归入会话组，同一人的内容按剧情时间轴从早到晚排列；包含最新一条消息的会话组排在最前。围绕林若岚的“今天还是你来处理吗”，玩家需要完成三轮回复：先追问是否见过或上一位处理人，再核对时间、处理内容、工号或派单记录，最后选择调取历史回访或按标准流程重新报修。对话只建立重复上门与记录缺失，不提前确认私人关系。

### 1.2 建立实际占用冲突

1. 在起始工单的“资料状态：待复核”区域打开 `1204 长期空置房巡检记录`；也可搜索 `1204` 或 `空置房` 找到同一档案。
2. 并列查看四张现场原图：防尘罩下留置的高价值家具、封存的立式空调、带有新鲜蔬菜及清洁水痕的厨房，以及门外童鞋。记录产权人陈大国、证件尾号4417、儿童床品和异常门禁。1204档案不显示“公开信息索引”，也不提示经侦关联；陈大国的公开新闻需要玩家主动用产权人姓名交叉搜索。
3. 四张图中只有左脚童鞋内露出的纸条可点击；其他照片、童鞋主体和图注都不响应点击。点击纸条后，儿童健康信息卡在影像组下方展开，并把拾获物 `FP-0713-26` 与该卡的只读副本写入档案。仅仅打开巡检档案不会揭示儿童姓名。
4. 展开的卡片提供儿童姓名与生日；监护人姓名只显示姓氏和掩码，完整姓名仍须从服务排班取得。之后仍可搜索 `童鞋`、`儿童健康` 或 `健康信息卡` 重开归档副本，但主流程不再要求额外搜索一次。
5. 搜索 `定时服务`、`服务排班`、`许建国` 或 `赵秀兰`。
6. 打开 `1号楼第二季度定时入户服务排班`。它包含多户无关服务记录，玩家需找到1204行。
7. 执行“跨表核验 / 服务授权与门禁”，取得 `vacancyMismatch`。
8. 为完整证据，可搜索 `陈大国` 或 `经侦通报` 阅读公开信息，但不要把媒体标题当作司法结论。

关键认知：许建国、赵秀兰过去是服务联系人；旧身份无法证明当前合法居住，但终止后的每日门禁证明1204并非空置。

完成实际占用冲突核验后，1204报警人、住户端、物业客服中心和安保值班会连续追加四条紧急消息，明确说明儿童失联、室内搜索无果、110已经受理以及消防通道与录像保全要求。事件首次升级时物业系统窗口会短促强烈抖动并出现红色边框；减少动态效果模式下不执行抖动。已经越过该节点的旧存档会补播一次新版紧急演出，写入已读状态后不再随刷新重复。该事件不是滴水投诉的原附件，也不能由打开普通工单直接触发。

### 1.3 受理失联事件并复核监控异常

1. 打开失联儿童紧急留言，分别询问“最后在哪里见到她？”和“报警回执是什么？”。
2. 得到最后目击：`2026-07-13 00:03`，1204次卧；回执：`DL-0713-0041`。
3. 接警事件受理后，安防中心才生成独立的公共区域录像保全任务。搜索 `监控`、`录像保全`、`DL-0713-0041` 或 `00:04`，打开 `12层公共区域事件回放复核`。
4. 播放串联视频；低音量拾音轨包含电梯门机、运行与停靠等公共区域设备环境声，不包含人声线索。音轨或视频播放失败时仍可用逐帧复核。
5. 只多选异常时间节点：`00:04`、`00:07`、`00:10`、`00:12`。
6. 提交后取得 `wetFootprints`。

四个节点的含义：

- `00:04`：1204门磁和潮湿双足痕迹。
- `00:07`：消防门/13层网关形成路径。
- `00:10`：回放丢帧或恢复节点。
- `00:12`：录像缓存出现重复序列；不是画面里出现了人。

只有执行“跨表核验 / 服务授权与门禁”并取得 `vacancyMismatch`，失联儿童紧急留言才触发。监控复核发生在失联事件之后，不能反过来作为事件的触发条件；单纯阅读文章仍不会触发。

### 1.5 下发五点搜救路线

搜索 `搜索路线`、`消防楼梯` 或 `失联儿童`，把候选现场图像拖入五个路线槽位并按顺序排列。已入路线的图像可在槽位间拖动换位，也可拖回候选区取消；触屏设备可点击图像增删，并用槽位上的前移、后移按钮调整：

1. 1204儿童房
2. 1204门外
3. 消防楼梯
4. 13层前室
5. 1304门外

电梯厅、1304室内和地库都有图像，但其事件、门禁或网关记录排除本次路线。三个排除项只显示图像、地点、时间和设备字段，不再用长段落重复描述“无人、无移动、地面干燥”等空画面结论。提交正确路线后触发救援场景演出：许芷遥先在1304门外相邻消防前室获救，并提及一名衣服全湿的小姑娘；镜头随后越过前室、沿湿痕移向始终没有开锁记录的1304，最后停在门板上若隐若现的顾长河身影。此末帧只用于恐怖演出，不写入监控证据，也不在此时显示顾长河姓名；减少动态效果模式直接展示末帧，玩家也可手动跳到最后一帧。

错误路线只返回证据不连续提示，不清空玩家已经编排的节点；玩家可以保留当前方案继续换位或替换图像。

此时真相推导中的CASE 01开放。章节只确认路线和策略失误，将引导者标记为“未核实”。

2. 核对用水曲线与顶面照片，取得 `zeroWater`。
3. 搜索 `声纹`、`儿童哼唱` 或 `浴缸滴水`，打开独立的 `1304 夜间声纹分轨报告`。
4. 播放四条来自实地录音的MP3声轨：管道水流低鸣、远处电视播报、浴缸滴水、女孩轻声哼唱儿歌。哼唱没有歌词，旋律断续，能听见自然换气与浴室短反射；播放器只显示可由听感确认的客观描述。

## 2. 1304：事故与注销账号

### 2.1 补齐家庭历史

1. 搜索 `顾长河` 或 `1304住户`，阅读重点回访。回访对象附件是一张右下角被剪去的旧身份证复印件；鼠标移动时，照片中的双眼会小幅追随玩家，但这只是恐怖演出，不写入证据。顾长河只主观陈述自己总听见已经分开的前妻敲门并打扰“我和孩子的生活”，该表述不能证明梁静宜实际到场。
2. 许芷遥获救后重新搜索 `1304`，检索结果会新增 `1304 墙面修补前影像记录`；也可使用正文中的 `身高刻度` 或 `小满` 找到它。救援前该影像不进入任何检索结果。
3. 搜索 `顾小满`、`浴缸` 或 `2021-08-21`，阅读历史事故附件。页面主体是一张2021年地方报纸的泛黄受潮扫描件，雨夜救援照片与标题只证明儿童送医这一外部报道事实；物业附件转写折叠在报纸下方，用于核对到场、移交和110补记的来源边界。
4. 搜索 `梁静宜` 或 `不在场`，阅读异地行程并执行三方凭证核验，取得 `wifeAlibi`。

玩家此时可以判断：小满真实存在；事故不只是普通漏水；梁静宜在顾长河死亡时不在现场。但不能把“想念”“宽恕”当作系统事实。

### 2.2 确认顾长河状态

满足以下条件后，`1304 账号主体状态异常复核` 开放：

- 许芷遥已获救。
- 声纹净化完成。
- 历史事故与梁静宜行程均已阅读。

条件全部满足后，重新执行首页的“确认1304户主状态”任务或搜索 `1304`，房号索引才会追加该复核表；条件不足时不提前显示受限标题。

照录公安回函：

- 人员状态：死亡
- 死因：玩家自行照录文本；填写内容同时包含“酒精”和“中毒”即可通过，例如“急性酒精中毒”或“因酒精中毒死亡”。输入框不提供答案选项。

提交后，顾长河注销账号在留言板发言。其触发条件不是随机时间，而是玩家确认死亡主体字段。

### 2.3 保全顾长河会话

1. 在留言板打开顾长河消息。
2. 可先引用公安回函或门禁/会话审计；两种回复都会让他意识到当前活动对象不是实体门禁。
3. 在保全会话前可使用“重新选择回复”查看另一组对话；之后选择“附加事故回执，保全会话并停用令牌”。
4. 取得 `fatherAware`；林若岚只提示先按姓名查周明川，不再直接发送密码。

### 2.4 核对1304死亡主体与异常账号链

真相推导CASE 02需要：许芷遥路径、顾长河死亡、异常留言令牌三组证据。

界面先明确复核目的：解释顾长河死亡、本人门禁停用后，1304为何仍与本次救援及留言写入有关。玩家从八条记录中为“历史事故、主体状态、凭证处置、本次关联、当前活动”五个阶段按顺序选择：

1. `A-1304-0821 / 110附件`
2. `公安协查回函 / 2023-02-08 00:36`
3. `DOOR-1304 / 2023-02-08 09:20`
4. `RESCUE-0713 / CAM-13F / 2026-07-13 00:07`
5. `MSG-1304 / TOKEN / 本次会话`

玩家可以逐条撤回选择；错误提交保留已经组装的核验链，并区分“记录选错”与“五类记录正确但阶段/顺序错误”。

章节演出随后呈现物业早已把事故、死亡主体和残留会话放入 `1304-FAMILY-KEEP`，并警告主角不得动用私情。

## 3. 1104：周明川与本地账号
