# 《不存在的住户》剧情真值表

> 完整剧透。本文面向创作和开发，不面向首次游玩的玩家。

## 一句话真相

主角不是在调查一栋闹鬼的公寓，而是在调查自己为什么死后仍以物业管理员 `CJ-0713` 的身份每天重新上班。

恒目及其背后的全知教会发现，意外死亡者的遗物、未结关系与重复的信息系统可以形成短暂的“回返窗口”。澄江物业被改造成观察设施：封存物是锚点，员工账号是容器，工单是唤醒路径，记忆一致性校正负责让实验在下一班次重新开始。

## 叙事层级

开发时必须区分三种陈述：

| 层级 | 含义 | 允许出现的位置 |
| --- | --- | --- |
| 故事真相 | 作者设定中真实发生的事 | 真相页、结局、本文 |
| 档案证实 | 玩家能用系统材料闭合的事实 | 游戏内档案、证据台账、章节演出 |
| 故意留白 | 材料不能证明，或不应替角色判断的部分 | 游戏内应明确保留不确定性 |

例如：顾小满确实引导许芷遥离开1204，这是故事真相；物业材料只能证明移动路线、两个低矮轮廓和许芷遥的陈述。游戏内不能把“鬼魂救人”写成工程结论。

## 核心人物

### CJ-0713 / CS-046 / 主角

- 真实身份：林若岚的丈夫。
- 生前状态：2025年11月4日22:31车祸死亡。
- 事故次日：同一实名哈希被HMO-ADMIN建立为 `CJ-0713`，没有劳动合同、面试、体检或审批。
- 当前表层身份：澄江物业的物业管理员，负责空置房巡检、住户回访和异常工单。
- 更早周期身份：客服坐席 `CS-046`。
- 循环特征：每天08:41打卡，00:10连接中断；251次有效打卡，0次有效下班。
- 叙事边界：游戏内只呈现T-04终端、连续质检号、相同停顿和被撤回的归因。只有真相页明确046与0713是同一意识。

### 林若岚

- 1404住户，主角的妻子。
- 在车祸中幸存，下肢重伤，使用轮椅。
- 将丈夫的骨灰转出并留在1404。
- 经历了223次被系统登记为“首次接触”的重逢，始终认得主角。
- 她知道主角会找糖、胃不好、如何调整轮椅脚踏，也知道他说谎前会先吸气。
- 坏结局中，她在第224次“首次接触”后不再纠正主角，也不再相信下一次会不同。

### 周明川 / ZM-0602

- 物业失联员工。
- 发现17名员工的无目的地“内部转移”、异常资金、ZC-LH标签规律和恒目通讯后遭清除。
- 遗体被封在1104西墙少掉的42厘米空腔内。
- 在手机同步摘要中留下 `11·04·2713`，用于打开1104联合记录。
- 其本地凭据由房号、墙体缺失厚度和人事状态修改次数组成：`1104-42-17`。
- 账号 `ZM-0602` 已注销，但仍可读取四份未同步的本地证据。

### 顾小满

- 顾长河与梁静宜的女儿，5岁，曾住1304。
- 2021年8月21日浴室事件中死亡。
- 故事真相：顾长河醉酒状态下的暴力与看护失职导致死亡。
- 遗留痕迹：身高刻度、浴缸滴水、孩童哼唱、湿衣影子。
- 故事真相：她在2026年7月13日引导许芷遥离开1204，沿消防楼梯到1304门外前室。
- 她救下另一个孩子不等于原谅父亲，也不能成为顾长河的赎罪证明。

### 顾长河

- 顾小满父亲，1304注销住户。
- 2021年事故当晚醉酒，涉及家庭暴力和看护失职。
- 2023年2月8日00:36死于急性酒精中毒。

- 他最终承认小满死亡时自己在门内。停止令牌只终止系统写入，不提供宽恕。

### 梁静宜

- 顾长河前妻、顾小满母亲，2021年后迁出。
- 要求维修时保留“小满 五岁”的身高刻度。
- 顾长河死亡时位于286公里外的康复机构。门禁、客运、支付和巡房记录构成完整不在场证明。

### 许建国、赵秀兰、许芷遥

- 许建国和赵秀兰原是1204定时入户服务联系人。
- 服务于2026年3月31日终止，但关联卡从4月3日起几乎每日通行。

## 四个案件

### CASE 01：1204空置房与失联儿童

档案可以证明：

- 台账标记空置，但存在新鲜食物、儿童床品、28码童鞋和持续门禁。
- 历史服务已经终止，关联卡仍进入1204。
- 鞋内健康卡、鞋盒日期与门禁都指向许芷遥在此生活。
- 1304水表零变化，1204顶面无渗漏，固定六分钟声响不是常规漏水。
- 公共画面、门磁、湿痕与手环网关闭合五点路线。
- 1304无开锁记录；许芷遥在门外相邻消防前室获救。

故事真相：顾小满以湿衣、赤脚和领先一个转角的影子引导许芷遥离开危险房间。

物业责任：`VACANT-CLOSE` 把生活痕迹自动归为“产权人临时存放物”，连续两次跳过实际占用复核。物业不是完全没看见，而是用规则决定不处理。

保留问题：许家为何住入1204、与陈大国有什么关系，现有材料没有答案。

### CASE 02：1304家庭事故与残留会话

档案可以证明：

- 2021年110联动附件记录未成年人看护风险、酒后暴力和浴室现场。
- 物业前台摘要只保留“浴室意外”。
- “小满 五岁”的身高刻度证明顾小满真实生活过。
- 顾长河2023年死亡，梁静宜有完整异地证明。
- 2026年写入系统的是留言令牌，而非顾长河门禁。
- 许芷遥获救陈述提到顾小满，将两次儿童事件连接起来。

故事真相：顾小满之死源于顾长河的暴力和失职；顾长河残留的会话承载否认、愧疚与迟来的承认。

物业责任：`1304-FAMILY-KEEP` 将死亡主体、儿童事故附件和残留会话保持关联，却不向前台暴露冲突，并警告CJ-0713“不得动用私情”。

保留问题：系统无法证明门外呼唤是思念，也不能登记成宽恕。

### CASE 03：1104失联员工与异常墙体

档案可以证明：

- 竣工图净宽4.80米，三次实测4.38米，西墙少42厘米。
- 插座孔附近高温、氨类和TVOC提示封闭空腔存在有机来源。
- 周明川的转移单没有车辆、目的地、接收部门、签收人或本人签字。
- HMO-ADMIN在三天内修改其状态17次。
- 处置记录早于人事状态和现场异常，说明清除流程预先存在。
- 公安破拆后，以工牌与DNA确认遗体为周明川。

故事真相：恒目驻场合规体系因周明川追查项目而将他灭口、封墙。材料能证明预谋清除链，但没有具体执行者姓名。

延伸事实：17份无目的地转移对应17名查到异常后被外派、离职或失联的员工，其中6人的最后门禁靠近后来封闭施工的房屋。

### CASE 04：1404与主角身份

档案可以证明：

- 事故死亡者哈希与CJ-0713实名哈希完全一致，紧急联系人指向1404。
- CJ-0713在事故次日被批量创建，没有正常入职附件。
- 林若岚将同名死者的封存物从殡仪馆转出并留在1404。
- 恒目将无电源、无芯片的封存物标为 `ZC-LH/CJ-0713`，绑定外部打卡终端。
- CJ-0713每天出现却从不下班；1404累计223次“首次接触”。
- 回访冷备份保留糖、轮椅、呼吸停顿与重复自我介绍。

故事真相：主角是林若岚已故的丈夫；骨灰是回返锚点；CS-046是更早清除周期中的主角。

物业责任：`MEM-CONSISTENCY` 在00:10清除关系和检索，把林若岚重新标成普通住户，把主角重新派成第一次上门的物业管理员。1404投诉甚至被转派给被投诉的CJ-0713本人。

## 恒目与全知教会

游戏内可证明的组织行为：

- 恒目管理顾问拥有物业数据清理、终端重置、账号状态改写和ZC-LH标签权限。
- “特殊保管服务”“终端校准”“数据过滤”等费用最终归集到恒目关联文化基金。
- 单眼图形早于当前企业备案，授权来源缺失。
- 培训、缓存清理、令牌重建和内部转移的时间高度重合。
- 周明川的通讯残片使用“观察者”“回返窗口”“记忆一致性”等非物业术语。

作者层真相：恒目是全知教会在物业、殡葬寄存和长期观察项目中的执行外壳。教会利用遗物和关系研究死者意识回返，并通过信息系统让观察对象重复进入可控岗位。

游戏内不要让组织直接自白，也不要把无法由附件证实的宗教机制写成审计结论。

## 结局

### 完整证据结局 / 离开

前置：周明川工程与人事证据完成，CS-046回访归档缺口完成，主角阻止记忆覆盖。

主角向警方和业委会提交封存物、1104空腔、回访缺口与资金审批链。证据先离开系统，主角的灵魂随后第一次越过物业边界。林若岚留在门内告别；00:10后CJ-0713不再回来。

### 循环结局 / 重新打卡

主角只完成当前工单，不继续追查恒目。系统将关系字段归零。次日08:41，他以“第一次上门”的物业管理员身份再次站在1404门口。

林若岚经历第224次首次接触。她不再纠正“第一次”，签下名字，倒掉第二杯水，收起糖盒。恐怖的核心不是跳吓，而是她终于不再等待丈夫想起来。

## 不得重新引入的错误

- 不把“1204卧室顶面渗漏排查”写成包含声纹分轨的同一份报告；声纹是独立安防档案。
- 不把水表和顶面湿度包装成不真实的“联合检测”。
- 不直接把许建国、赵秀兰写成1204住户或授权居住人。
- 不把陈大国的媒体报道写成司法定罪。
- 不让许芷遥改成不随父姓的无解释姓名。
- 不把00:12描述成画面里出现人物；它的异常是重复序列/缓存校验。
- 不把监控缺失位置改成纯文字“无画面”；排除地点也有实际图像。
- 不在普通档案中提前写明顾小满、顾长河、CS-046或主角身份的完整答案。
- 不把物业警告写成作者替玩家总结真相；警告应体现压制和知情。
- 不把顾小满救人写成“家庭团圆”“原谅父亲”或替顾长河洗清责任。
